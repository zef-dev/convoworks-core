Yes No processor

A simple yes/no junction processor. This processor matches against "yes", "no", and variations thereof ("yep", "sure", "uh huh", "nope", "nah", etc.)

When to use?

If you're posing a simple yes/no question to the user, instead of having to mix and match simple processors and intents, you can use this predefined processor.

How to use:

  1. In the Yes flow, add elements you wish to execute if the user gives an affirmative response.
  2. Conversely, add whichever elements you want if the user gives a negative answer in the No flow.
