Element Queue

This element executes its child elements in sequence. You can decide how often it will check if they've been executed, i.e. once per request, session, or installation.

Optionally, you may also put elements in a "Done" flow that will run if the Queue element is read again, but has already executed its elements in the given scope.

You can also set the element to always re-read its children, without checking if it's been run already.

When to use?

When you have elements you wish to run in sequence, while having control over how often they run, and with an additional flow in case they've been executed.

How to use?

  • Place whichever children elements you wish to execute in sequence in the Elements flow.
  • Optionally, you may also add child elements to the Done flow which will run if the Queue element is read again, but has already executed its children.
  • The Should reset property dictates whether or not to execute the Done flow once the queue has been executed, or to re-read the queue again.
  • Scope type governs how "often" the queue will run, if Should reset is toggled off. This essentially means, "read the queue once per installation/session/request".
