Media Block

Serves for handling media playing requests (they are sessionless).

Overview

Audio players on the smart speaker devices are working a bit differently than the regular conversation services. Once you initiate the audio stream your service stops and the audio player takes control.
While in that mode the user can take just a few different actions, by voice or by tapping on the screen. Users can ask for the Next or Previous song, they can turn the loop and shuffle on and off. All these commands are handled automatically by the Media Block.

While executing the child flows (next not available, ...) you have access to the media info which you can use to enhance your responses. E.g. "There are ${media_info.count} songs found and you are on the last one."

You also have access to the last media info in the entire service which you can use to enhance your resume functionalities.

Here are some example how to use the last media info:
  • Get count of found songs since the playback has started - ${last_media_info.count}
  • Is last song since the playback has started - ${last_media_info.last}
  • Is first song since the playback has started - ${last_media_info.first}
  • Number of song that is playing since the playback has started - ${last_media_info.song_no}
  • Index of song that is playing since the playback has started - ${last_media_info.song_no - 1}
  • Last known loop status since the playback has started - ${last_media_info.loop_status}
  • Last known shuffle status since the playback has started - ${last_media_info.shuffle_status}
  • Was playing - ${last_media_info.playing}
  • Current song artist since the playback has started - ${last_media_info.current.artist}
  • Current song title since the playback has started - ${last_media_info.current.songTitle}
  • Current file url since the playback has started - ${last_media_info.current.fileUrl}
  • Current song image url since the playback has started - ${last_media_info.current.songImageUrl}
  • Current song background image url since the playback has started - ${last_media_info.current.songBackgroundUrl}
  • Next song artist since the playback has started - ${last_media_info.next.artist}
  • Next song title since the playback has started - ${last_media_info.next.songTitle}
  • Next file url since the playback has started - ${last_media_info.next.fileUrl}
  • Next song image url since the playback has started - ${last_media_info.next.songImageUrl}
  • Next song background image url since the playback has started - ${last_media_info.next.songBackgroundUrl}
